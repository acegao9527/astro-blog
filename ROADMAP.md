# 路线图

## 当前重点

- [ ] OG 图片打通
  验收标准：文章页输出 `og:image` 和 `twitter:image`，优先使用文章 `cover`，无封面时回退到站点默认图。
- [ ] 文章页目录逻辑瘦身
  验收标准：目录侧栏以 CSS `sticky` 为主，减少定位计算和内联脚本依赖，同时保留移动端可用性。
- [ ] 图片优化链路重构
  验收标准：减少普通 `<img>` 直出，建立更稳定的封面图与正文图片处理策略，为后续图片优化留出接口。

## 已完成

- [x] 配置与文档收口
  结果：统一 `BLOG_DIR` / `SITE_URL` 配置来源，新增 `.env.example`，移除隐式默认值，README 与结构文档同步到真实实现。
- [x] 首页列表改成静态分页
  结果：首页固定第一页，新增 `/page/[page]/` 路由，移除“查看更多”客户端追加逻辑。

## 待办

- [ ] 公众号极简技术文样式优化
  目标：把博客文章页逐步打磨成接近优质公众号技术长文的阅读体验，风格关键词为极简、克制、学术感、技术研究风。详细路线图见 [`docs/公众号极简样式优化路线图.md`](./docs/公众号极简样式优化路线图.md)。
- [ ] 内容同步脚本测试
  目标：为 `scripts/sync-posts.mjs` 增加最小回归保护，覆盖 slug 冲突、缺失字段和素材改写等关键场景。
- [ ] 中文 slug 路径策略
  目标：评估是否继续保留当前中文路径编码结果，或统一为更稳定的 ASCII slug 方案。
- [ ] 统一封面图约定
  目标：明确 `cover`、`hero`、`ogImage` 的职责，避免 frontmatter 能力和页面消费能力继续分叉。
- [ ] 评论模块
  目标：为文章页增加轻量、稳定、低维护的评论能力，优先保证静态站点兼容性、加载性能和垃圾评论可控性。详细路线图见 [`docs/评论模块路线图.md`](./docs/评论模块路线图.md)。

## 工作流

- 每完成一项任务，立即把对应条目从 `[ ]` 改成 `[x]`，并补一句结果说明。
- 如果任务不再继续，改成 `[~]` 并写明原因，避免待办长期失真。
- 新任务优先补充到 `当前重点` 或 `待办`，不要散落到其他文档里。
